# 12주차 블록체인 보안 실습 과제

- **테스트 환경:** 로컬 Hardhat (메인넷 배포 절대 안 함)
- **실행 방법:** `npm test` 치면 자동으로 해킹 시나리오랑 방어 코드 다 돌아감.

## 1. DAO 재진입 공격 (Part 1)
- 돈을 빼는 함수에서 잔액을 깎기 전에 외부로 돈을 먼저 보내버리는 게 문제였음.
- 방어: 돈을 먼저 깎고(Checks-Effects-Interactions) 송금하도록 구조를 수정함.

## 2. Parity 1차 공격 (주인 탈취)
- 지갑 라이브러리에 주인을 정하는 initWallet 함수가 있는데, 이걸 아무나 부를 수 있어서 털림.
- 방어: 생성자(constructor)에서 미리 주인을 세팅하거나, 한 번만 초기화되도록 막음.

## 3. Parity 2차 공격 (자폭)
- 해커가 라이브러리 주인이 된 다음 selfdestruct(자폭)를 시켜버림. 
- 라이브러리가 증발하니까 멀쩡한 지갑들도 코드를 못 찾아서 돈이 묶여버림(Freeze).
- 방어: 그냥 자폭 기능을 코드에서 없애버리는 게 제일 안전함.

## 첨부 자료
- `logs/` 폴더: 내 컴퓨터에서 npm test 돌려서 나온 성공 로그들.
- `diagrams/` 폴더: delegatecall 메모리 충돌 다이어그램.
